# 作品名称

基于 RDK X5 的智能无影灯系统

## 摘要

本作品面向学习、书写与桌面作业等近距离照明场景，设计并实现了一套基于 `RDK X5` 的智能无影灯系统。系统以 `RDK X5` 作为嵌入式主控平台，结合 `D435` 相机、麦克风、机械臂式灯体执行机构和 `ROS2 Humble` 中间件，构建了“视觉感知、语音交互、决策协调、执行控制”一体化闭环。系统运行过程中，视觉模块实时采集桌面图像，完成手部位置检测、纸面阴影区域识别和使用者坐姿状态分析；语音模块支持离线唤醒词识别和固定指令解析，可实现跟踪模式启停、暖光模式切换和冷光模式切换；协调模块综合视觉状态与语音指令，生成机械臂与灯光控制命令；执行模块驱动机械臂调整灯具指向，使照明区域主动跟随手部活动并削弱书写阴影，从而提升近桌面作业的舒适性与稳定性。

`RDK X5` 平台具备 `8` 核 `ARM Cortex-A55` 处理能力和 `10 TOPS BPU` 端侧智能推理能力，能够为系统提供稳定的嵌入式计算基础，并为后续更复杂的视觉模型板端部署提供良好的硬件条件。相比传统固定角度台灯，本系统具备环境感知、自主决策和动态补光能力，能够根据实际阴影分布自动调整照明方向，同时通过语音交互降低用户操作门槛，并结合姿态提醒功能扩展健康学习辅助能力。作品兼顾了嵌入式实现、机器人控制与智能感知融合，适用于智慧学习桌面、办公工位、创客实验平台及人机协同照明等应用场景，具有较好的展示性、工程完整性和后续扩展价值。

## 第一部分 作品概述

### 1.1 功能与特性

本系统围绕“主动消除手部书写阴影”这一核心目标展开设计，具备桌面视觉感知、语音交互、机械臂联动调光和坐姿提醒四类主要功能。系统通过相机实时检测手部位置与纸面阴影分布，判断当前是否存在遮挡光照的局部暗区，并输出阴影相对手部的方向信息；当检测到需要补光时，系统自动控制灯体朝合适方向偏转，以降低纸面阴影面积。与此同时，用户可以通过“你好小灯”等唤醒词发出模式切换指令，实现跟踪启停和灯光色温切换。针对长时间学习场景，系统还可结合人体关键点信息识别身体歪斜、头部过近等不良姿态，并进行语音提醒。系统具有感知实时、交互自然、结构清晰和扩展方便等特点。

[图1 系统功能展示图]

### 1.2 应用领域

本作品适用于需要长时间近距离照明且对阴影遮挡较敏感的应用场景。在学生书写、绘图、手工制作等场合，传统台灯通常只能提供固定方向照明，当手部进入作业区域后容易在纸面形成明显阴影，影响观察与书写质量。本系统能够针对桌面局部阴影进行主动感知与动态补偿，适用于智慧学习桌、实验操作台和办公书写工位等场景。对于创客教育、机器人课程和嵌入式系统教学，本作品还可作为融合视觉、语音、机械臂和边缘智能的综合实验平台，展示多模块协同工作方式。若进一步与专用灯具、姿态监测和个性化调光策略结合，也可扩展到儿童健康照明、辅助书写、桌面作业安全照明等领域。

### 1.3 主要技术特点

本系统的主要技术特点体现在嵌入式平台融合、模块化软件架构和面向场景的控制策略三个方面。首先，在平台层面，系统采用 `RDK X5` 作为核心控制单元，依托其 `8` 核 `A55` 处理器、Ubuntu 22.04 系统环境、`ROS2 Humble` 机器人软件生态和 `10 TOPS BPU` 推理能力，实现了感知、决策与执行的统一部署。其次，在软件架构层面，系统使用 `ROS2` 对视觉、语音、协调、灯控和机械臂控制进行模块划分，模块间通过标准消息接口解耦，便于调试、复用和后续升级。最后，在算法层面，系统不是简单跟踪手的位置，而是将手部位置与纸面阴影区域进行联合分析，利用阴影相对手部的几何关系生成补光方向，使灯光调节更符合真实书写需求。此外，系统语音交互采用离线识别方式，避免对网络环境的依赖，更适合嵌入式部署。

### 1.5 主要创新点

1. 提出面向桌面书写场景的手部与阴影联合感知方法，不以手部跟踪代替补光控制，而是直接围绕阴影消除目标进行决策。
2. 将视觉感知、语音交互、姿态提醒和机械臂执行统一部署在 `RDK X5` 嵌入式平台上，形成完整的端侧智能照明闭环。
3. 采用 `ROS2` 模块化架构构建系统，便于感知算法、控制策略和硬件执行单元独立扩展。
4. 预留 `BPU` 模型迁移能力，可支持后续将更复杂的视觉模型加速部署到板端。

### 1.6 设计流程

系统设计流程遵循“需求分析、方案拆解、嵌入式部署、联调验证”的工程思路。首先针对书写场景中的阴影遮挡问题，确定“视觉检测阴影、机械臂主动补光、语音辅助交互”的总体目标；随后完成主控平台、相机、语音和执行机构的功能划分，并以 `ROS2` 建立统一通信框架；在此基础上实现手部检测、阴影检测、语音命令识别与姿态提醒等功能模块，再通过协调节点完成多模块状态融合，最终在 `RDK X5` 上完成板端集成与系统联调，形成可运行的智能无影灯原型系统。

## 第二部分 系统组成及功能说明

### 2.1 整体介绍

本系统由硬件系统和软件系统两部分构成，整体采用“感知层、决策层、执行层”三级结构。感知层包括 `D435` 相机与麦克风，用于采集桌面视觉信息和用户语音命令；决策层以 `RDK X5` 为核心，运行视觉处理程序、语音处理程序、系统协调节点和姿态提醒逻辑，通过 `ROS2` 话题实现各功能模块之间的信息交换；执行层包括智能灯具、机械臂驱动链路和灯光模式控制单元，用于根据决策结果调整照明方向和工作模式。系统工作时，相机持续发布包含手部中心、阴影中心、阴影面积、姿态状态等信息的视觉状态消息，语音模块发布用户指令消息，协调节点结合系统模式决定是否发布机械臂控制命令与灯光控制命令，最终完成桌面阴影补偿和交互反馈。

[图3 系统整体框图]

在通信结构上，系统设置了 `/vision/state`、`/voice/command`、`/system/mode`、`/arm/command` 和 `/light/command` 等核心话题。视觉模块和语音模块作为上游信息源，系统协调模块处于核心中枢位置，负责将用户意图和环境状态映射为执行命令；机械臂控制模块则把高层角度调整量转换为底层关节速度命令，驱动灯体完成动态偏转。这种结构层次清晰，既保证了系统实时性，也提高了可维护性与可扩展性。

### 2.2 硬件系统介绍

#### 2.2.1 硬件整体介绍

本系统硬件部分主要由 `RDK X5` 主控板、`D435` 深度相机、USB 麦克风、六轴机械臂灯体、LED 光源模组以及供电与连接附件组成。其中，`RDK X5` 负责运行 Ubuntu 22.04、`ROS2 Humble` 和各类感知控制程序，是整个系统的嵌入式计算与控制中心；`D435` 相机用于采集桌面书写区域的图像信息，为手部与阴影检测提供输入；麦克风负责采集用户语音命令；机械臂用于承载灯体并调整照射方向；LED 光源提供照明输出。`RDK X5` 具备 `8` 核 `ARM Cortex-A55` 和 `10 TOPS BPU`，既能满足当前系统的多任务协同运行需求，也支持后续将更复杂的检测与分割网络部署到板端进行加速推理。

[图4 硬件整体连接图]

#### 2.2.2 机械设计介绍

机械部分采用“机械臂 + 灯具末端执行器”的结构设计。机械臂基座安装在桌面或支撑底座上，通过多关节串联形成可调节的空间姿态控制能力，末端安装照明灯头，使光源能够随着目标区域变化而改变入射角。系统当前主要利用机械臂的水平偏航和俯仰方向调节能力完成补光动作，从而在不影响用户书写的前提下实现动态减影。整体机械结构强调稳定、紧凑和可重复定位，既适合比赛展示，也有利于后续向更高自由度的精确光照控制扩展。

#### 2.2.3 电路各模块介绍

电路部分包括主控供电模块、传感器接口模块、语音采集模块、机械臂控制接口和灯光驱动接口。主控供电模块为 `RDK X5` 提供稳定工作电源，并为相机、麦克风和控制外设分配电源；传感器接口模块完成相机与主控之间的数据连接；语音采集模块通过 USB 音频设备向系统提供本地离线语音输入；机械臂控制接口负责连接舵机驱动或现有机械臂控制器，接收来自主控的运动指令；灯光驱动接口实现亮度、色温或开关状态控制。通过这种模块化硬件设计，系统既可在现有比赛原型上稳定运行，也便于后期对灯具驱动电路和专用控制板进行独立优化。

### 2.3 软件系统介绍

#### 2.3.1 软件整体介绍

软件系统基于 `ROS2 Humble` 构建，采用节点化、模块化设计思想，主要包括视觉感知模块、语音交互模块、系统协调模块、机械臂控制模块和灯光控制模块。各模块在 `RDK X5` 上统一运行，并通过自定义消息接口进行通信。视觉模块负责从相机图像中提取手部位置、阴影区域和姿态状态；语音模块负责离线识别唤醒词和固定命令；系统协调模块负责模式管理、消息融合和控制决策；机械臂控制模块负责将高层控制量转换为底层关节指令；灯光控制模块则根据工作模式调整光照状态。系统既可独立运行单个模块进行调试，也可整体启动形成完整的智能无影灯功能链路。

[图9 软件系统总体架构图]

#### 2.3.2 软件各模块介绍

1. 视觉感知模块

视觉感知模块是系统的核心输入模块，主要完成手部检测、阴影检测、阴影关联和姿态检测。系统首先读取 `D435` 彩色图像，在图像中检测人手中心位置；随后结合纸面亮区域提取和相对暗区筛选方法识别阴影候选区域，并优先选择距离手部最近的阴影块作为当前补光目标；在此基础上计算阴影中心、阴影面积、是否需要补光以及阴影相对手部的方向向量，最终封装为 `VisionState` 消息发布。为了兼顾学习健康场景，视觉模块还集成了姿态检测逻辑，通过人体关键点分析肩部倾斜、身体偏移和头部过近等问题，为后续提醒提供依据。

[图10 视觉处理流程图]

`VisionState` 消息包含手部中心坐标、阴影中心坐标、阴影面积、补光建议点、姿态状态和姿态评分等关键字段，能够为协调层提供完整的感知输入。

2. 语音交互模块

语音交互模块采用本地离线识别方案，核心流程包括音频采集、语音识别、唤醒词检测和固定命令解析。系统通过麦克风采集用户语音，并使用离线语音识别引擎将音频转写为文本，随后判断文本中是否包含唤醒词“你好小灯”；若检测到唤醒词，则进一步匹配“开启跟踪模式”“关闭跟踪模式”“切换暖光模式”“切换冷光模式”等固定命令，并发布 `VoiceCommand` 消息。该方案无需依赖外部网络，响应稳定，部署简单，适合嵌入式端侧语音交互需求。

[图11 语音处理流程图]

3. 系统协调模块

系统协调模块负责对视觉状态和语音指令进行统一管理，是整个软件系统的决策中枢。模块订阅 `/vision/state` 和 `/voice/command` 话题，并根据当前模式决定系统是否进入自动跟踪补光状态。当跟踪模式开启且检测到有效手部与阴影时，模块根据阴影中心相对手部中心的偏移量计算期望的偏航与俯仰控制量，并生成 `ArmCommand` 消息；若没有有效阴影或系统处于关闭状态，则输出保持或空闲命令。同时，该模块会根据姿态检测结果启动提醒状态机，对持续性不良坐姿进行语音提醒，从而增强系统的人机友好性。

[图12 协调决策流程图]

4. 机械臂控制模块

机械臂控制模块负责将协调层给出的高层方向命令映射为底层关节运动控制量。模块接收 `ArmCommand` 中的 `yaw` 和 `pitch` 调整量，并按照设定比例映射到机械臂底座关节和俯仰关节的实时速度指令，再通过 `/servo_node/delta_joint_cmds` 发送 `JointJog` 消息以驱动执行端运动。该控制方式结构简单、实时性好，适合本系统以“灯具朝向调整”为核心任务的应用需求。

[图13 机械臂控制流程图]

5. 灯光控制模块

灯光控制模块负责管理照明模式、亮度和色温参数。系统通过 `LightCommand` 消息描述灯光开关状态、目标亮度、色温和照明目标位置，可根据用户语音命令切换暖光与冷光模式，也可在自动补光过程中与机械臂动作联动，维持舒适照明效果。该模块为系统后续扩展更复杂的灯具硬件驱动、电流调光和环境自适应策略预留了明确接口。

[图14 灯光控制流程图]

5. 嵌入式扩展能力突出。依托 `RDK X5` 的 `10 TOPS BPU`，系统后续可继续部署更高精度的检测、分割和姿态估计模型，具备良好的升级空间。

## 第四部分 总结

### 4.1 可扩展之处

本系统在现有基础上仍具有较强的扩展空间。首先，在视觉侧可进一步将更高精度的手部检测、阴影分割和人体姿态估计模型转换为适配 `RDK X5` 的 `BPU` 推理模型，以提升复杂场景下的识别精度与实时性；其次，在执行侧可引入更精细的灯头姿态控制与闭环位置反馈，实现从“关节方向代理控制”向“末端灯光空间最优控制”升级；再次，在灯光侧可加入亮度自适应、环境光检测和个性化色温调节功能，使系统更贴近真实学习照明需求；最后，在交互侧可扩展更丰富的本地语音命令和状态播报能力，增强系统的自然交互体验。通过这些扩展，系统有望进一步发展为具有产品化潜力的智能桌面照明平台。

### 4.2 心得体会

本作品的完成过程是一次典型的嵌入式智能系统综合实践。相比单一算法验证，本系统更强调在真实平台上把视觉、语音、机器人执行和中间件通信整合成一个可以稳定运行的整体。项目实施过程中，我们深刻体会到嵌入式系统设计不仅是某一个算法模块的实现，更重要的是系统级架构的划分、软硬件接口的明确以及多模块联调时的稳定性控制。以 `RDK X5` 为核心平台开展开发，使我们能够在一个统一的边缘智能环境中完成感知、推理和控制任务，也让我们更加理解 `ROS2` 模块化通信架构在复杂机器人系统中的价值。

在视觉部分，我们认识到书写场景下真正需要解决的问题并不是简单检测手，而是理解手与阴影之间的关系。只有将目标从“看见手”转变为“消除阴影”，系统的控制策略才真正具有场景意义。在语音部分，我们体会到离线识别在嵌入式应用中的现实价值，本地命令识别虽然功能边界清晰，但稳定、可控、部署简洁，非常适合比赛原型和实际设备场景。在机械臂控制部分，我们通过高层方向量到关节速度控制量的映射，建立了从感知结果到物理动作之间的直接联系，也对机器人执行链路的实时性和鲁棒性有了更深理解。

更重要的是，本项目让我们看到多学科融合设计的价值。一个看似简单的“台灯”系统，在加入嵌入式主控、边缘智能感知、语音交互和机械臂控制后，已经转变为一个完整的人机协同智能装备原型。未来若进一步引入 `BPU` 模型加速、灯具定制结构和更完善的健康辅助策略，该系统不仅能够作为竞赛作品，也有望面向智慧学习桌面、儿童辅助书写和人机协作照明等方向继续发展。整个研发过程提升了我们在嵌入式平台应用、系统工程设计和智能硬件集成方面的实践能力，也为后续开展更复杂的边缘智能机器人项目打下了良好基础。
